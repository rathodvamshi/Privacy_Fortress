Two-Locker Architecture

Locker 1: Ephemeral

Session-based vault

  • Storage: Redis with TTL (30 minutes default)
  • Scope: Per session only
  • Contains: Token-to-PII mappings ([NAME_1] → Alice)
  • Lifecycle: Created on first message, auto-expires
  • Wiped: When session ends or TTL expires
  • Purpose: Enable real-time unmasking during chat
  • Security: Never persisted, RAM-only operations

Locker 2: Persistent

User profile vault

  • Storage: MongoDB with AES-256-GCM encryption
  • Scope: Per user (one profile only)
  • Contains: Encrypted blob {name, college, email}
  • Lifecycle: Created only with user consent
  • Wiped: When user clicks "Forget Me"
  • Purpose: Enable cross-device sync and profile recreation
  • Security: Fully encrypted at rest, decrypted in RAM only

🔑 Key Principle

No Session Vault Sprawl: We never store full session vaults permanently. We keep ONE encrypted profile per user and recreate session mappings from it when needed. This keeps the design simple, compliant, and auditable.

Five Security Layers

1

Frontend Security

RAM-only state management. No PII in localStorage or cookies. State cleared on page refresh or logout. Zero persistence on client side.

2

Mask-Maker Pipeline

Triple-layer detection: spaCy NER for entities, Regex for patterns (email, phone), RapidFuzz for typos and variations. Catches 99%+ of common PII.

3

Ephemeral Vault

Redis-based session mappings with TTL. Auto-expiry ensures session data never outlives the conversation. Keys are salted per session.

4

Persistent Vault

AES-256-GCM encrypted profile storage. Each user has ONE encrypted blob. Decryption happens in RAM only. "Forget Me" permanently deletes the profile.

5

LLM Shield

Prompt injection detection. Response validation. AI never sees real PII. Model can be swapped without changing privacy guarantees.

Key Features

PII Detection & Masking

Automatically detects names, emails, colleges, companies, and phone numbers using a combination of NER, regex patterns, and fuzzy matching. Handles typos and variations.

AES-256-GCM Encryption

Military-grade encryption for all persistent storage. Each user profile is encrypted as a single blob. Decryption happens only in RAM, never written to disk unencrypted.

TTL Auto-Delete

Session tokens automatically expire after 30 minutes (configurable). No manual cleanup needed. Ephemeral vault self-destructs, preventing data accumulation.

Prompt Injection Shield

Protection against LLM jailbreak attempts and prompt injection attacks. Validates inputs and outputs to ensure the AI stays within safe boundaries.

Zero-Knowledge LLM

The AI never sees real user identities. All PII is replaced with tokens before reaching the model. Cannot be trained on or leak real personal information.

Cross-Device Sync

With user consent, encrypted profile enables seamless experience across devices. One profile is decrypted and recreated into session state. No session vault duplication.

API Endpoints

POST /api/chat

Send a message to the AI. PII is automatically detected, masked, sent to LLM, then unmasked in the response. Stored masked in database.

GET /api/sessions

List all chat sessions for the authenticated user. Messages are stored in masked format and unmasked on retrieval.

PUT /api/vault/consent

Enable "Remember me" / cross-device sync. Creates encrypted profile from current session or explicit fields.

GET /api/vault/profile

Get profile metadata (has_profile, consent status). Does not return decrypted data without explicit request.

PUT /api/vault/profile

Save or update encrypted profile. Can extract from current session or accept explicit name, college, email fields.

DELETE /api/vault/profile

"Forget me" — permanently delete encrypted profile and clear all ephemeral session vaults for the user.
